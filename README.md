# Feature Scaling

The project applies:

### StandardScaler

to normalize numerical features.

---

## Why Scaling Matters

KNN is a distance-based algorithm.

Without normalization:

- Large-scale features dominate
- Distance calculations become biased
- Model performance decreases

Example:

```text
MonthlyIncome → 1000–20000
Age → 18–60
```

Without scaling, income would overpower age in Euclidean distance calculations.

---

## Standardization Formula  ```math z = \frac{x - \mu}{\sigma} ```



where:

- μ = feature mean
- σ = standard deviation

This transforms all features to:

```text
Mean = 0
Standard deviation = 1
```

# K-Nearest Neighbors (KNN)

The first machine learning model implemented is:

### K-Nearest Neighbors Classification

---

# KNN Principle

KNN predicts class labels based on:

```text
Nearest neighboring data points
```

using Euclidean distance.

---

# Euclidean Distance Formula  ```math d(x,y)=\sqrt{\sum_{i=1}^{n}(x_i-y_i)^2} ```


The algorithm:

1. Computes distance to all points
2. Finds K nearest neighbors
3. Applies majority voting
4. Predicts attrition class

---

# K Optimization

The project evaluates:

```text
K = 1 → 20
```

to find the optimal value.

---

## Optimal K Value

```text
Best K = 7
```

with highest test accuracy.

---

# Bias–Variance Trade-Off

| K Value | Behavior |
|---|---|
| Low K | High variance, overfitting |
| Medium K | Balanced generalization |
| High K | High bias, underfitting |

# Model Comparison

The project demonstrates an important ML trade-off:

---

## KNN

Advantages:

- Higher overall accuracy
- Lower false positives
- Better majority-class prediction

Disadvantages:

- Poor minority-class recall
- Misses many attrition cases

---

## Gaussian Naive Bayes

Advantages:

- Better recall for attrition cases
- Captures more at-risk employees
- Better minority-class sensitivity

Disadvantages:

- Lower overall accuracy
- Higher false-positive rate

---

# Business Interpretation

In HR analytics:

```text
False Negative:
Employee leaves but model predicts stay
```

is usually more expensive than:

```text
False Positive:
Employee predicted to leave but stays
```

Therefore:

### Recall becomes critically important.

This explains why Gaussian Naive Bayes may still be useful despite lower accuracy.
